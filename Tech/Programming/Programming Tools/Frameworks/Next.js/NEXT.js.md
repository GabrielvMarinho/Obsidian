A [[React]] [[Framework]] for [FrontEnd](obsidian://open?vault=Obsidian&file=Tech%2FSoftware%2FConcepts%2FFRONT-END%2FFrontEnd) development

In nextJS you have the client side and the server side, the server side is very relevant to [[SEO]] so there should be as many as possible components in the server side, however, when it comes to functionality and routing, the component needs to be in the client side, so you should have especific buttons in the client side only

In next when using [[Hooks (React)]] you can either put the usestate in a input text component or you can add a onchange attribute to the input and add this usestate in the more generic component, like a filter composed of different input text's and pass the function in the onchange
